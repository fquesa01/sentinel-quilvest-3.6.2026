Which case the data belongs to

Which custodians the data belongs to

8️⃣ COMPLETE REPLIT PROMPT (copy/paste into Vibe Coding)
Create a new React/Next.js UI component set called 
“Connect New Source” inside /app/cases/[caseId]/sources/.

It must implement:

1. A main panel showing connected sources:
   - Query from /api/cases/[caseId]/sources
   - Cards: Slack, Dropbox, M365, Box, Clio
   - Buttons: Sync Now, Settings, Disconnect, Add Source

2. Add Source modal with 3 steps:
   STEP 1 — Provider selection grid 
     Providers: Slack, Microsoft 365, Google, Dropbox, Box, Clio, Teams, Zoom Chat, WhatsApp, Custom S3.
     Each provider card triggers STEP 2.

   STEP 2 — OAuth authentication page
     Button: “Connect <Provider>”
     On success, auto-advance to STEP 3.

   STEP 3 — Configuration:
     - Case mapping dropdown (pre-filled with caseId)
     - Custodian mapping table:
       Detected user -> suggested custodian from /api/custodians/resolve
     - Scope selection:
       For Slack: list channels
       For Dropbox: list folders
       For Google: list mailboxes/folders
       For Box: list folders
     - Sync frequency dropdown
     - Final review page with summary

3. On Finish:
   - POST /api/connectors to create connectorConfiguration row
   - Immediately POST /api/connectors/:id/sync
   - Redirect to updated Connected Sources panel
