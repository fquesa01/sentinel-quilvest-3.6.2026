2) NEW COMMUNICATIONS HEATMAP MODULE – SCOPE
------------------------------------------------

Create a new module called “Communications Heatmap” for each CASE that provides:

A) Time-of-Day / Day-of-Week Heatmap
- Grid: x-axis = hour (0–23), y-axis = day of week (Mon–Sun).
- Cell value = count of messages in that bucket, filtered by:
  - Case
  - Optional filters: employee/person, organization/domain, internal vs external, date range.
- Color scale: low volume → dark, high volume → bright.
- Used to spot:
  - Normal working patterns
  - Off-hours spikes
  - Busy days.

B) Person-to-Person Interaction Heatmap
- Matrix where rows = people, columns = people.
- Cell value = number of messages between those two in the selected period (sum of sent + received).
- Hover tooltip: show counts (sent, received, total) and most common domains.
- Ability to:
  - Filter by case, date range.
  - Limit to top N people by volume.
- Clicking a cell opens a side panel listing:
  - Recent messages between those parties.
  - “Add as Party” buttons for each person (reuse existing action).

C) Organization Interaction Heatmap
- Matrix where rows = organizations/domains (gmail.com, usawaterpolo.org, morganlewis.com, etc.).
- Columns = organizations/domains.
- Cell value = number of cross-domain messages.
- Used for:
  - Vendor monitoring
  - Law firm interaction
  - Crossover between internal and external domains.

D) Topic vs Person Heatmap (optional but scaffold it)
- Assume we can tag messages with topics (e.g., “contracts”, “HR”, “safety”, “legal”, “finance”).
- Heatmap where rows = topics, columns = people (or vice versa).
- Cell value = number of messages for that topic/person combo.
- For now:
  - Implement backend stubs that accept a topic field.
  - Add mock topic assignment logic (e.g., based on keyword search in subject/body) so the UI is demonstrable.

E) Anomaly / Change Heatmap
- For each person and/or domain, compute month-over-month (% change) in message volume.
- Display a simple heatmap where:
  - Rows = people or domains
  - Columns = months
  - Cell value = volume OR % change.
- Flag anomalies where volume deviates significantly from the baseline.
- Surface a list of “Notable Changes” in a side panel.

3) ANALYTICS & SCORING
------------------------------------------------

For each person (per case and globally), compute and store:

- engagementScore:
    weighted combination of:
      - totalMessages
      - uniqueContacts
      - ratio of sent vs received
      - internal vs external mix.

- responsivenessScore:
    based on average response time
    (delta between a received message and the next reply in the thread from that person).

- crossDomainRiskScore:
    higher when:
      - large share of messages are external.
      - interacting with many distinct external domains.

- topicRiskScore (stub):
    higher if:
      - topics include sensitive terms like “lawsuit”, “settlement”, “kickback”, “complaint”, etc.

- anomalyScore:
    derived from deviation from their own historical volume
    (e.g., z-score or simple % difference vs trailing 3-month average).

Create a backend aggregation job / service that can:
- Rebuild these scores for a case on demand.
- Optionally run periodically or when new data is ingested.
