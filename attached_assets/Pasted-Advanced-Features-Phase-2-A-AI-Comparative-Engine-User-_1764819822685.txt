“Show where the documents contradict Frank’s testimony.”

“Identify which parts of Frank’s interview are corroborated by the emails.”

“Which witness is least credible and why?”

“List all statements about the December 1 event and synthesize a timeline.”

Contradiction tables

Narrative timelines

Heatmaps of agreement vs conflict

The engine does:

Element-by-element analysis

Whether facts satisfy or contradict elements

Missing evidence

Required next steps

Exposure assessment

Recommendations

Primary theory

Alternative theories

5. User Workflows
Workflow 1: Creating Findings While Reviewing Interviews

User opens an interview →
Clicks “Add to Findings” →
Modal opens:

Title

Note type

Findings text

Auto-insert transcript snippet

Link to specific timestamp

User saves.
Sentinel auto-links it back to the interview.

Workflow 2: Comparing Witnesses

User selects two interviews →
Clicks “AI Compare” →
Sentinel generates:

Points of agreement

Points of contradiction

Missing information

Credibility analysis

Suggested follow-up questions

User can click “Save to Findings.”

Workflow 3: Legal Application

User navigates to Findings →
Button:

Apply Law to Facts

Dropdown of legal domains.
User selects:

Title VII

Antitrust

Harassment

Fraud

Contract breach

Insurance coverage

Etc.

AI produces:

Element-by-element chart

Evidence matrix

Missing proof

Exposure levels

Recommendations

User can save entire output as a standalone Finding entry.

Workflow 4: Building the Final Report

Once Findings are complete, user clicks:

Generate Report from Findings

AI compiles:

Summary

Factual findings

Credibility findings

Legal conclusions

Recommended action

Supporting citations

Editable like Google Docs.
