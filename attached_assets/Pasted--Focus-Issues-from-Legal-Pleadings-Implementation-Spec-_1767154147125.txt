# Focus Issues from Legal Pleadings - Implementation Spec

## Overview

Allow users to upload legal pleadings (complaints, motions, answers) to automatically extract key factual allegations that become Focus Issues for Ambient Intelligence monitoring. During meetings, the AI compares live testimony against these allegations and flags evidence that either **supports** or **hurts** the user's position.

## Position Classification Logic

### For Plaintiff (proving allegations):

| Evidence Type | Classification | Meaning |
|---------------|----------------|---------|
| Confirms allegation | 👍 SUPPORTS | Helps prove the case |
| Contradicts allegation | 👎 HURTS | Undermines the case |
| Shows pattern/notice | 👍 SUPPORTS | Strengthens the case |
| Defendant denial | 👎 HURTS | Creates issue of fact |

### For Defendant (defending against allegations):

| Evidence Type | Classification | Meaning |
|---------------|----------------|---------|
| Contradicts allegation | 👍 SUPPORTS | Helps defense |
| Confirms allegation | 👎 HURTS | Undermines defense |
| Shows no pattern | 👍 SUPPORTS | Weakens plaintiff's case |
| Admission by client | 👎 HURTS | Damaging to defense |

### Classification Prompt

```typescript
async function classifyEvidence(
  document: Document,
  allegation: Allegation,
  side: 'plaintiff' | 'defendant',
  liveStatement?: string
): Promise<PositionClassification> {
  const prompt = `
    You are analyzing evidence in a legal proceeding.
    
    ALLEGATION (${allegation.paragraph}):
    "${allegation.text}"
    
    EVIDENCE:
    Source: ${document.source}
    Content: "${document.content}"
    ${liveStatement ? `\nLIVE TESTIMONY just given: "${liveStatement}"` : ''}
    
    USER REPRESENTS: ${side.toUpperCase()}
    
    Analyze whether this evidence SUPPORTS or HURTS the ${side}'s position 
    regarding this specific allegation.
    
    Consider:
    - Does it confirm or contradict the factual claims in the allegation?
    - Does it strengthen or weaken the legal argument?
    - Would this be good or bad news for the ${side}?
    
    Return JSON:
    {
      "position": "supports" | "hurts",
      "confidence": "high" | "medium" | "low",
      "explanation": "One sentence explaining why this supports/hurts the position"
    }
  `;
  
  return await llm.classify(prompt);
}
```

## Extraction Algorithm

### Step 1: Parse Pleading Structure

```typescript
async function parsePleading(text: string, type: string): Promise<ParsedPleading> {
  // Identify paragraph markers: ¶, numbered paragraphs, "COUNT I", etc.
  const paragraphPattern = /(?:¶\s*\d+|^\d+\.|^[A-Z]+\s+[IVX]+)/gm;
  
  // Split into sections
  const sections = splitBySections(text, paragraphPattern);
  
  return {
    type,
    sections,
    rawText: text
  };
}
```

### Step 2: Extract Factual Allegations

```typescript
async function extractAllegations(
  parsedPleading: ParsedPleading
): Promise<Allegation[]> {
  const prompt = `
    Analyze this ${parsedPleading.type} and extract the KEY FACTUAL ALLEGATIONS 
    that would need to be proven or defended at trial.
    
    Focus on:
    - Specific facts alleged (dates, times, actions, knowledge)
    - Who did what and when
    - What the defendant knew or should have known
    - Causation claims
    - Damages claims
    
    DO NOT extract:
    - Boilerplate jurisdictional allegations
    - Incorporation by reference paragraphs
    - Prayer for relief
    - Legal conclusions without factual basis
    
    Document:
    ${parsedPleading.rawText}
    
    Return JSON array:
    [
      {
        "paragraph": "¶ 12",
        "text": "Full text of the allegation",
        "type": "negligence|notice|damages|causation|breach|other",
        "keywords": ["key", "terms", "to", "search"]
      }
    ]
  `;
  
  const extracted = await llm.extract(prompt);
  
  return extracted.map(a => ({
    id: generateId(),
    ...a,
    active: true,
    createdAt: new Date()
  }));
}
```

### Step 3: Generate Search Queries

```typescript
function generateSearchQueries(allegation: Allegation): SearchQuery[] {
  // Create boolean search variations
  const queries: SearchQuery[] = [];
  
  // Exact keyword combinations
  queries.push({
    type: 'keywords',
    query: allegation.keywords.join(' AND ')
  });
  
  // Date-focused search
  const dates = extractDates(allegation.text);
  if (dates.length > 0) {
    queries.push({
      type: 'temporal',
      query: dates.map(d => `date:${d}`).join(' OR ')
    });
  }
  
  // Person-focused search
  const persons = extractPersonNames(allegation.text);
  if (persons.length > 0) {
    queries.push({
      type: 'person',
      query: persons.map(p => `from:${p} OR to:${p} OR mentions:${p}`).join(' OR ')
    });
  }
  
  // Semantic search using full allegation text
  queries.push({
    type: 'semantic',
    query: allegation.text
  });
  
  return queries;
}
```

## Real-Time Monitoring

### Modified Ambient Intelligence Loop

```typescript
async function ambientMonitoringLoop(session: Session) {
  const config = await getSessionFocusConfig(session.id);
  
  setInterval(async () => {
    // Get latest transcript chunk
    const transcriptChunk = await getLatestTranscript(session.id, 20); // 20 seconds
    
    if (!transcriptChunk || transcriptChunk.length === 0) return;
    
    // Extract claims from live testimony
    const claims = await extractClaims(transcriptChunk);
    
    // For each active allegation, search for relevant evidence
    for (const allegation of config.allegations.filter(a => a.active)) {
      // Generate search query combining allegation keywords + transcript
      const query = buildFocusedQuery(allegation, claims);
      
      // Search case RAG
      const results = await searchCaseRag(session.caseId, query);
      
      // Classify each result
      for (const result of results) {
        const classification = await classifyEvidence(
          result,
          allegation,
          config.side,
          transcriptChunk
        );
        
        // Store and emit result
        const evidenceResult: EvidenceResult = {
          id: generateId(),
          sessionId: session.id,
          allegationId: allegation.id,
          sourceType: result.type,
          sourceId: result.id,
          sourceTitle: result.title,
          sourceDate: result.date,
          excerpt: result.excerpt,
          position: classification.position,
          confidence: classification.confidence,
          explanation: classification.explanation,
          triggerTranscript: transcriptChunk,
          surfacedAt: new Date()
        };
        
        await saveEvidenceResult(evidenceResult);
        
        // Emit to client
        socket.emit('evidence:found', evidenceResult);
        
        // Special alert for "hurts" with high confidence
        if (classification.position === 'hurts' && classification.confidence === 'high') {
          socket.emit('evidence:alert', {
            type: 'hurts_position',
            evidenceResult
          });
        }
      }
    }
  }, 20000); // Every 20 seconds
}
```
