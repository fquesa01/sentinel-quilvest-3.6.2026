## 7. BACKEND / DATA MODEL CHANGES

### New DB table: data_lake_items
- id, user_id, source (outlook | gmail | onedrive | gdrive | upload), item_type (email | pdf | docx | xlsx | other), name, file_path (object storage), indexed_at, last_synced, file_size, gemini_indexed (boolean), metadata (JSON)

### New DB table: data_lake_connectors
- id, user_id, connector_type, status (connected | disconnected), access_token (encrypted AES-256-GCM), refresh_token (encrypted), token_expiry, last_sync, items_indexed, live_sync_enabled

### Ingestion pipeline (reuse/extend existing):
- On file upload: run OCR if needed → chunk → embed → store in Gemini RAG index tagged with user_id and source
- On email sync: parse email headers/body → chunk → embed → store in Gemini RAG index
- All items tagged with user_id so searches are scoped to the individual user only
