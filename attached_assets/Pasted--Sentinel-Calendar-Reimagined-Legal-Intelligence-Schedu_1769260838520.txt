# Sentinel Calendar: Reimagined Legal Intelligence Scheduling

## Executive Summary

A calendar system that looks and feels like Microsoft Outlook but operates with the intelligence of Motion, the flexibility of Calendly, and the legal-specific features that small law firms and PE firms desperately need. This isn't just a calendar — it's a **Legal Time Intelligence System**.

---

## Core Philosophy

**"Outlook Familiarity, AI Intelligence, Legal Precision"**

Small law firms and PE firms don't have time to learn new interfaces. They want something that feels familiar but works smarter. The Sentinel Calendar delivers:

1. **Zero Learning Curve** — Outlook-style interface they already know
2. **AI That Adapts** — Dynamic scheduling that respects legal workflows
3. **Matter-Centric** — Everything links to clients, matters, and billable time
4. **Privilege-Protected** — All AI interactions under attorney-client privilege

### 2. AI-POWERED SCHEDULING (Motion/Reclaim Innovation)

#### Intelligent Auto-Scheduling
```
Traditional: You manually find time for tasks
Sentinel: AI schedules tasks around your existing commitments
```

**How it works:**
1. Create a task with deadline, duration, and priority
2. AI analyzes your calendar, energy patterns, and workload
3. Automatically places task in optimal time slot
4. Dynamically reschedules if conflicts arise

#### Focus Time Protection
- Set daily/weekly focus time goals (e.g., "4 hours deep work daily")
- AI automatically blocks and defends these hours
- Smart "flex" blocks that move when high-priority meetings arise
- "Do Not Disturb" periods that sync with phone/desktop

#### Smart Rescheduling
- When meetings change, dependent tasks automatically shift
- Missed tasks roll to next available slot
- Deadline-aware: Tasks approaching due dates get higher priority
- Energy-aware: Complex work scheduled during peak productivity hours (configurable)

#### AI Meeting Prep
- 15 minutes before any external meeting:
  - Auto-surface relevant documents from CRM/DMS
  - Show matter history and recent communications
  - Prepare brief of attendee relationships
  - Generate suggested talking points

### 3. EXTERNAL BOOKING SYSTEM (Calendly Innovation)

#### Scheduling Links
```
https://sentinel.counsel/book/john-smith/consultation
```

**Features:**
- Multiple event types (15min call, 30min consultation, 60min strategy session)
- Custom availability per event type
- Buffer time between meetings
- Daily/weekly booking limits
- Minimum advance notice requirements
- Custom branding per attorney

#### Routing Forms (Lead Qualification)
Before booking, prospects answer screening questions:
```
1. What type of legal matter? [dropdown]
2. Is this time-sensitive? [yes/no]
3. Brief description of your situation [text]
4. Preferred contact method [phone/video/in-person]
```

Based on answers:
- Route to correct attorney (practice area matching)
- Route to first available (round-robin)
- Flag high-priority for immediate follow-up
- Reject and redirect if outside practice areas

#### Round-Robin Scheduling
- Distribute client bookings evenly across team
- Weight by availability, specialization, current workload
- Respect attorney preferences (no evenings, no Fridays)

#### Payment Collection at Booking
- Integrate with Stripe/PayPal
- Collect consultation fees upfront
- Retainer deposits
- Automatic invoice generation

#### Meeting Polls (Group Scheduling)
- Send poll to multiple parties
- Everyone selects available times
- Auto-book the time that works for majority
- Perfect for depositions, mediations, closings

### 5. LEGAL-SPECIFIC INNOVATIONS

#### Matter-Linked Events
Every calendar event can be linked to:
- **Client** (from CRM)
- **Matter** (case/deal)
- **Billing code** (activity type)
- **Documents** (from DMS)

This enables:
- One-click time entry after events
- Automatic billing descriptions
- Matter timeline reconstruction
- Conflict checking

#### Billable Time Tracking Integration
```

│ 10:00 AM - 11:30 AM (1.5 hours)        │
```

After meeting ends:
- Popup prompts for time confirmation
- Auto-populates billing entry
- Option to adjust duration
- Add detailed description
- Submit directly to billing system

#### Court Deadline Integration
- Sync with court filing systems
- Auto-calculate derivative deadlines
  - "Motion due 30 days from filing"
  - "Response due 14 days from motion"
  - "Hearing date + travel time"
- Deadline cascade views
- Malpractice insurance requirements compliance

#### Deposition/Hearing Scheduler
**Special event type with unique fields:**
- Witness name
- Court reporter booked
- Videographer booked
- Exhibits prepared (checklist)
- Notice served (with date)
- Venue/remote platform
- Opposing counsel notified
- Auto-generate deposition notice document

#### Statute of Limitations Tracker
- Create SOL events with countdown
- Automatic 30/60/90 day warning alerts
- Link to intake matter
- Cannot be deleted without supervisor approval
- Audit trail of all modifications

#### Conflict Calendar
- Before accepting any matter/meeting:
  - Auto-check for party conflicts
  - Show related matters
  - Flag potential ethical issues
- Integration with conflicts database

### 7. WORKFLOW AUTOMATION

#### Pre-Meeting Workflows
```
WHEN: 24 hours before client meeting
THEN:
  1. Send reminder email to client
  2. Generate prep brief
  3. Pull recent communications
  4. Alert attorney if documents missing
```

#### Post-Meeting Workflows
```
WHEN: Meeting ends
THEN:
  1. Send follow-up email template
  2. Create time entry draft
  3. Schedule follow-up if requested
  4. Update matter status
```

#### Deadline Workflows
```
WHEN: 7 days before filing deadline
THEN:
  1. Alert responsible attorney
  2. Alert supervising partner
  3. Check document status
  4. Block day before for final prep
```

## Technical Architecture

### Database Schema (Drizzle ORM)

```typescript
// Calendar Events
export const calendarEvents = pgTable('calendar_events', {
  id: uuid('id').defaultRandom().primaryKey(),
  organizationId: uuid('organization_id').references(() => organizations.id),
  createdById: uuid('created_by_id').references(() => users.id),
  
  // Basic Event Data
  title: varchar('title', { length: 500 }).notNull(),
  description: text('description'),
  location: varchar('location', { length: 500 }),
  videoConferenceUrl: varchar('video_conference_url', { length: 1000 }),
  
  // Timing
  startTime: timestamp('start_time').notNull(),
  endTime: timestamp('end_time').notNull(),
  allDay: boolean('all_day').default(false),
  timezone: varchar('timezone', { length: 100 }).default('America/New_York'),
  
  // Recurrence (RFC 5545 RRULE format)
  recurrenceRule: varchar('recurrence_rule', { length: 500 }),
  recurrenceEndDate: timestamp('recurrence_end_date'),
  parentEventId: uuid('parent_event_id').references(() => calendarEvents.id),
  
  // Classification
  eventType: varchar('event_type', { length: 50 }).notNull(), // meeting, task, deadline, focus, personal
  visibility: varchar('visibility', { length: 20 }).default('default'), // public, private, confidential
  status: varchar('status', { length: 20 }).default('confirmed'), // tentative, confirmed, cancelled
  
  // Legal Intelligence Links
  clientId: uuid('client_id').references(() => contacts.id),
  matterId: uuid('matter_id').references(() => matters.id),
  
  // Billing Integration
  isBillable: boolean('is_billable').default(false),
  billingActivityCode: varchar('billing_activity_code', { length: 50 }),
  estimatedDuration: integer('estimated_duration_minutes'),
  actualDuration: integer('actual_duration_minutes'),
  billingRate: decimal('billing_rate', { precision: 10, scale: 2 }),
  timeEntryId: uuid('time_entry_id').references(() => timeEntries.id),
  
  // AI Scheduling
  isAiScheduled: boolean('is_ai_scheduled').default(false),
  canBeRescheduled: boolean('can_be_rescheduled').default(true),
```
