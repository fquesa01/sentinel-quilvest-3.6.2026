Assume we already have:

1.1 Add analytics helpers (e.g., in `analytics/communication.py`)

Implement functions that work over both lists:

- `get_participant_key_from_email(email_msg) -> list[str]`
  - Derive participant IDs from email addresses, e.g. `"email:john@example.com"`.
  - Include `from_` and recipients.

- `get_participant_key_from_chat(chat_msg) -> str`
  - Use `ChatParticipant.id` and/or phone number, e.g. `"chat:+13051234567"`.

- `build_communication_summary(email_messages, chat_messages) -> list[dict]`
  Each dict:
  - `participant_id: str`
  - `display_name: str | None`
  - `channels: list[str]`  # e.g. ["email", "whatsapp", "sms_ios"]
  - `sent_count: int`
  - `received_count: int`
  - `total_count: int`
  - `percent_of_total: float`
  - `first_activity_at: datetime | None`
  - `last_activity_at: datetime | None`

- `build_communication_matrix(email_messages, chat_messages) -> dict`
  Return:
  ```python
  {
    "participants": [
      {"id": "email:john@example.com", "display_name": "John"},
      ...
    ],
    "matrix": [
      # 2D array, rows = sender index, cols = receiver index
      # each cell: {"total": int, "email": int, "chat": int}
    ],
    "max_total": int  # max value in any cell, for heatmap scaling
  }

Logic:


For each email:


Sender = from_


Receivers = all in to + cc + bcc


Increment sender→receiver edges.




For each chat message:


Sender = sender_id


Receivers = all other participants in the conversation.


Increment sender→receiver edges.




Track whether counts came from email or chat for each cell.


build_unified_timeline(email_messages, chat_messages) -> list[dict]
Output a list sorted by timestamp:
{
  "id": str,
  "channel_type": "email" | "chat",
  "channel_subtype": "email" | "whatsapp" | "sms_ios" | "sms_android" | "other_chat",
  "sent_at": datetime.isoformat(),
  "sender_display": str,
  "recipients_display": list[str],
  "subject_or_preview": str,
  "has_attachments": bool,
  "direction": "inbound" | "outbound" | "unknown"
}



For email:


channel_type = "email", channel_subtype = "email"


subject_or_preview = subject if present else first 100 chars of body_text.




For chat:


channel_type = "chat", channel_subtype from source_type


subject_or_preview = first 100 chars of text.




1.2 FastAPI endpoints (in main.py)
Add:


GET /analytics/communication-summary


Optional query params: min_messages (int, default 1)


Uses build_communication_summary.


Filters out participants with total_count < min_messages.


Sort by total_count descending.




GET /analytics/communication-matrix


Uses build_communication_matrix.


Optional top_n (default 10) to limit to top N participants by total_count before building matrix. This keeps the heatmap manageable.


Return structure:
{
  "participants": [...],
  "matrix": [...],
  "max_total": 42
}





GET /timeline


Optional query params: channel_type ("all" | "email" | "chat"), limit (int, default 200).


Uses build_unified_timeline, filters by channel_type, sorts by sent_at descending, then applies limit.




All endpoints return JSON suitable for direct consumption by the frontend.
