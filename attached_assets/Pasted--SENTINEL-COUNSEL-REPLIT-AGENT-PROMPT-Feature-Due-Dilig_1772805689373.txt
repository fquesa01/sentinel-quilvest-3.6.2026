## FEATURE OVERVIEW

Build a **Due Diligence Checklists** module that supports two distinct investment tracks:

1. **Equity DD Checklist** — for PE buyouts, M&A, minority investments, co-investments
2. **Debt DD Checklist** — for senior secured loans, mezzanine, revolving credit, ABL, direct lending

Each checklist is:
- Pre-populated with institutional-grade items organized by category
- Fully trackable (status, priority, responsible party, date received, data room path, notes)
- Linked to a specific **Deal** in the deal workspace
- Exportable to Excel and PDF
- AI-augmented (Ava can analyze completion gaps and surface risks)

## DATABASE SCHEMA

Add the following tables to the existing Prisma schema:

```prisma
// ─── DUE DILIGENCE CHECKLISTS ──────────────────────────────────────

model DDChecklist {
  id             String          @id @default(cuid())
  dealId         String
  deal           Deal            @relation(fields: [dealId], references: [id], onDelete: Cascade)
  track          DDTrack         // EQUITY or DEBT
  title          String          // e.g., "Project Heartland - Equity DD"
  status         ChecklistStatus @default(ACTIVE)
  createdById    String
  createdBy      User            @relation(fields: [createdById], references: [id])
  createdAt      DateTime        @default(now())
  updatedAt      DateTime        @updatedAt
  items          DDChecklistItem[]
  
  @@index([dealId])
}

enum DDTrack {
  EQUITY
  DEBT
}

enum ChecklistStatus {
  ACTIVE
  ARCHIVED
  COMPLETED
}

model DDChecklistItem {
  id               String          @id @default(cuid())
  checklistId      String
  checklist        DDChecklist     @relation(fields: [checklistId], references: [id], onDelete: Cascade)
  itemNumber       Int             // Sequential within checklist
  section          String          // e.g., "A. Corporate & Governance"
  sectionCode      String          // e.g., "A"
  itemText         String          @db.Text
  priority         DDPriority      @default(MEDIUM)
  status           DDItemStatus    @default(OPEN)
  responsibleParty String?
  dateRequested    DateTime?
  dateReceived     DateTime?
  dataRoomPath     String?
  notes            String?         @db.Text
  isBestPractice   Boolean         @default(false)
  isCustom         Boolean         @default(false)   // User-added items
  displayOrder     Int
  createdAt        DateTime        @default(now())
  updatedAt        DateTime        @updatedAt

  @@index([checklistId])
  @@index([status])
  @@index([priority])
}

enum DDPriority {
  HIGH
  MEDIUM
  LOW
}

enum DDItemStatus {
  OPEN
  IN_PROGRESS
  PARTIAL
  PROVIDED
  NOT_APPLICABLE
}

// Add to existing DDTemplate model (or create if not exists)
model DDChecklistTemplate {
  id          String        @id @default(cuid())
  track       DDTrack
  name        String        // e.g., "Standard PE Buyout - Equity DD"
  description String?
  isDefault   Boolean       @default(false)
  sections    Json          // Array of section objects with items
  createdAt   DateTime      @default(now())
  updatedAt   DateTime      @updatedAt
}
```

## API ENDPOINTS

Add the following REST endpoints to the Express backend:

```
// Checklist CRUD
POST   /api/deals/:dealId/dd-checklists            — Create checklist from template for a deal
GET    /api/deals/:dealId/dd-checklists            — List all checklists for a deal
GET    /api/dd-checklists/:checklistId             — Get checklist with all items
DELETE /api/dd-checklists/:checklistId             — Archive checklist

// Item Management
PATCH  /api/dd-checklist-items/:itemId             — Update item (status, responsible party, notes, dates, path)
PATCH  /api/dd-checklists/:checklistId/bulk-update — Bulk update multiple items
POST   /api/dd-checklists/:checklistId/items       — Add custom item
DELETE /api/dd-checklist-items/:itemId             — Delete custom item only

// Progress & Analytics
GET    /api/dd-checklists/:checklistId/progress    — Completion stats by section and priority
GET    /api/dd-checklists/:checklistId/gaps        — Return all OPEN or IN_PROGRESS HIGH priority items

// Export
GET    /api/dd-checklists/:checklistId/export/excel  — Download as .xlsx
GET    /api/dd-checklists/:checklistId/export/pdf    — Download as .pdf

// Templates
GET    /api/dd-checklist-templates                 — List available templates
```
