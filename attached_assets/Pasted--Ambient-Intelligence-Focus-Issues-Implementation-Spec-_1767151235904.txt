# Ambient Intelligence Focus Issues - Implementation Spec

## Overview

Enhance the Ambient Intelligence feature to allow users to define specific "Focus Issues" that direct the AI's attention during meetings. Instead of surfacing all related documents, the system prioritizes and organizes results around the user's strategic objectives - particularly valuable during depositions where attorneys need to develop or contradict specific factual points.

## Core Concept

**Current behavior:** Every 20 seconds, extract keywords from transcript вҶ’ run boolean search across case RAG вҶ’ surface all matching documents

**Enhanced behavior:** Every 20 seconds, extract keywords from transcript вҶ’ for each active Focus Issue, run targeted boolean search with issue context вҶ’ tag and score results by issue relevance вҶ’ highlight contradictions and supporting evidence

## Relevance Classification

Each surfaced document should be tagged with one of:

| Tag | Icon | Color | Meaning |
|-----|------|-------|---------|
| CONTRADICTS | вҡ пёҸ AlertTriangle | Red | Document conflicts with what was just said |
| SUPPORTS | вң“ CheckCircle | Green | Document corroborates what was just said |
| PATTERN | вҡЎ Zap | Amber | Document shows recurring pattern/similar incidents |
| IMPEACHES | рҹ”„ RefreshCw | Red | Prior statement by same speaker that differs |
| RELATED | рҹ“„ FileText | Gray | Generally relevant but no clear support/contradiction |

### Contradiction Detection Logic

When processing transcript chunks, the AI should:

1. **Extract claims** - Identify factual assertions (times, dates, actions, knowledge)
2. **Compare to documents** - For each claim, check if case documents contain conflicting information
3. **Flag contradictions** - When document contradicts claim, tag with specific explanation

Example:
```
Transcript: "I sent him an email right away, probably around 7:40"
Document: Email timestamp shows 7:42 AM
Miller's testimony: "I received the email around 9:15 AM"

вҶ’ Flag Miller testimony as CONTRADICTS with note: 
  "Miller claims email received at 9:15 AM, but email timestamp is 7:42 AM"
```

## Search Algorithm Enhancement

### Current Algorithm (simplified)

```
every 20 seconds:
  transcript_chunk = get_last_20_seconds_of_transcript()
  keywords = extract_keywords(transcript_chunk)
  boolean_query = build_boolean_query(keywords)
  results = search_case_rag(case_id, boolean_query)
  display(results)
```

### Enhanced Algorithm

```
every 20 seconds:
  transcript_chunk = get_last_20_seconds_of_transcript()
  claims = extract_claims(transcript_chunk)  // New: identify factual assertions
  keywords = extract_keywords(transcript_chunk)
  
  if mode == 'focused':
    for each active_issue in focus_issues:
      # Combine issue context with transcript keywords
      issue_query = build_focused_query(
        keywords=keywords,
        issue_title=active_issue.title,
        issue_keywords=active_issue.keywords
      )
      
      results = search_case_rag(case_id, issue_query)
      
      for each result in results:
        # Classify relevance to the claim
        relevance = classify_relevance(
          document=result,
          claims=claims,
          issue=active_issue
        )
        
        result.focusIssueId = active_issue.id
        result.relevance = relevance.type
        result.relevanceNote = relevance.explanation
        
      display_grouped_by_issue(results)
  
  else:  # 'all' mode
    boolean_query = build_boolean_query(keywords)
    results = search_case_rag(case_id, boolean_query)
    
    # Still classify relevance even in all mode
    for each result in results:
      relevance = classify_relevance(result, claims)
      result.relevance = relevance.type
      
    display_flat_list(results)
```

### Focused Query Building

```typescript
function buildFocusedQuery(
  transcriptKeywords: string[],
  issueTitle: string,
  issueKeywords?: string[]
): BooleanQuery {
  // Extract key concepts from issue title
  const issueConcepts = extractConcepts(issueTitle);
  
  // Combine: (transcript_keywords) AND (issue_concepts OR issue_keywords)
  return {
    must: transcriptKeywords,
    should: [...issueConcepts, ...(issueKeywords || [])],
    boost: {
      // Boost documents that match issue keywords
      issueKeywords: 2.0,
      // Boost recent documents
      recency: 1.5
    }
  };
}
```

### Relevance Classification

```typescript
async function classifyRelevance(
  document: Document,
  claims: Claim[],
  issue?: FocusIssue
): Promise<RelevanceResult> {
  // Use LLM to classify
  const prompt = `
    Given this document excerpt:
    "${document.preview}"
    
    And these claims from the current testimony:
    ${claims.map(c => `- ${c.text}`).join('\n')}
    
    ${issue ? `Focus issue: ${issue.title}` : ''}
    
    Classify the relationship:
    1. Does the document CONTRADICT any claim? If so, explain specifically.
    2. Does the document SUPPORT any claim? If so, explain specifically.
    3. Does the document show a PATTERN of similar incidents?
    4. Is this a prior statement by the same speaker that differs (IMPEACHES)?
    5. Or is it just generally RELATED?
    
    Return JSON: { type, explanation }
  `;
  
  return await llm.classify(prompt);
}
```
