# Sentinel Counsel LLP - Enterprise E-Discovery Enhancement Prompt

## Project Context

I have an existing legal compliance and e-discovery platform called Sentinel Counsel LLP built with:
- **Frontend**: React + TypeScript + TailwindCSS + shadcn/ui
- **Backend**: Node.js/Express with PostgreSQL (Neon serverless) + Drizzle ORM
- **AI**: OpenAI GPT-5, Gemini embeddings, ElevenLabs transcription
- **Auth**: Replit Auth + Microsoft OAuth 365
- **Storage**: Object storage for documents

The platform currently handles case management, document review, communication ingestion (17 email formats, 9+ chat formats), AI-powered analysis, and compliance monitoring.

I need to add comprehensive enterprise-grade e-discovery capabilities. Please implement the following features in phases, ensuring each phase is fully functional before moving to the next.

### 2.2 Continuous Active Learning (CAL) Workflow

**Implementation requirements:**

Create a CAL service that:
1. Starts with seed set of ~200 documents coded by senior reviewer
2. Trains initial model using document embeddings (Gemini) + coded relevance
3. Ranks all uncoded documents by predicted relevance
4. Serves highest-ranked documents to reviewers
5. Retrains model after every N documents coded (configurable, default 50)
6. Tracks recall/precision metrics in real-time
7. Suggests stopping point when recall target achieved (typically 75-80%)

**API Endpoints:**
```
POST /api/cases/:caseId/tar/models - Create new TAR model
POST /api/cases/:caseId/tar/models/:modelId/train - Trigger training
GET /api/cases/:caseId/tar/models/:modelId/predictions - Get ranked documents
POST /api/cases/:caseId/tar/models/:modelId/code - Submit coding decision
GET /api/cases/:caseId/tar/models/:modelId/metrics - Get current metrics
POST /api/cases/:caseId/tar/models/:modelId/control-set - Create control set
GET /api/cases/:caseId/tar/models/:modelId/elusion-test - Run elusion test
```

**UI Components:**
- TAR Dashboard showing model status, metrics, training progress
- Prioritized Review Queue sorted by relevance score
- Real-time recall/precision charts
- Control set management panel
- Elusion test results with statistical validation
- "Gain Curve" visualization showing review efficiency

### 2.3 Control Set & Validation

**Implementation requirements:**
- Random sample selection for control set (statistically valid size based on population)
- Blind coding of control set (reviewers don't know it's control)
- Automated recall/precision calculation from control set
- Elusion testing: sample from documents predicted non-relevant, measure actual relevance
