# 1. MODULE OVERVIEW

## 1.1 Purpose

The Enterprise Risk Management (ERM) module transforms Sentinel Counsel into a comprehensive GRC platform capable of competing with IBM OpenPages, MetricStream, ServiceNow GRC, Diligent, and NAVEX.

**Core Capabilities:**
- **Visibility**: Complete view of all organizational risks across business units, geographies, and risk categories
- **Quantification**: Financial impact modeling using Monte Carlo simulation and VaR
- **Monitoring**: Real-time KRI tracking with automated alerting
- **Governance**: Risk appetite framework with cascading tolerance levels
- **Resilience**: Business continuity and disaster recovery planning
- **Intelligence**: AI-powered risk identification and predictive analytics

## 1.2 Target Users

| User Role | Primary Functions |
|-----------|-------------------|
| Chief Risk Officer (CRO) | Enterprise-wide oversight, board reporting, appetite setting |
| Risk Managers | Risk identification, assessment, monitoring, reporting |
| Business Unit Leaders | Operational risk ownership, control attestation |
| Internal Audit | Risk-based audit planning, control testing |
| Compliance Officers | Regulatory risk monitoring, compliance mapping |
| Executive Leadership | Dashboard consumption, decision support |
| Board/Audit Committee | Governance oversight, fiduciary review |

## 1.3 Integration with Existing Sentinel Modules

| Existing Module | ERM Integration Points |
|-----------------|----------------------|
| Investigations | Convert incidents to risks; Link risks to cases |
| Business Transactions | M&A risk assessments; Due diligence risks |
| eDiscovery | Evidence collection; Litigation risk tracking |
| Compliance Monitoring | Regulatory requirements; Compliance status |
| Communications Analysis | Detect emerging risks from communication patterns |
| Case Management | Risk event cases; Remediation tracking |

## 3.1 Core Risk Tables

```sql
-- =====================================================
-- RISK TAXONOMY & CATEGORIES
-- =====================================================

CREATE TABLE risk_categories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    parent_category_id UUID REFERENCES risk_categories(id),
    level INTEGER DEFAULT 1, -- 1=Top, 2=Sub, 3=Detailed
    sort_order INTEGER DEFAULT 0,
    color VARCHAR(20),
    icon VARCHAR(100),
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Seed data for standard risk taxonomy
INSERT INTO risk_categories (code, name, description, level) VALUES
('STRATEGIC', 'Strategic Risk', 'Risks affecting long-term organizational objectives', 1),
('OPERATIONAL', 'Operational Risk', 'Risks arising from internal processes, people, and systems', 1),
('FINANCIAL', 'Financial Risk', 'Risks affecting financial position and performance', 1),
('COMPLIANCE', 'Compliance Risk', 'Risks of regulatory violations and legal exposure', 1),
('REPUTATIONAL', 'Reputational Risk', 'Risks affecting organizational reputation and brand', 1),
('TECHNOLOGY', 'Technology Risk', 'Risks related to IT systems, cybersecurity, and data', 1),
('THIRD_PARTY', 'Third-Party Risk', 'Risks from vendors, suppliers, and business partners', 1),
('ESG', 'ESG Risk', 'Environmental, Social, and Governance risks', 1);


-- =====================================================
-- ORGANIZATIONAL STRUCTURE
-- =====================================================

CREATE TABLE business_units (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(200) NOT NULL,
    parent_unit_id UUID REFERENCES business_units(id),
    level INTEGER DEFAULT 1,
    unit_type VARCHAR(50), -- 'division', 'department', 'region', 'subsidiary'
    head_user_id UUID REFERENCES users(id),
    is_active BOOLEAN DEFAULT true,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);


-- =====================================================
-- ENTERPRISE RISK REGISTER
-- =====================================================

CREATE TABLE risks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    risk_id VARCHAR(50) UNIQUE NOT NULL, -- e.g., "RISK-2025-0001"
    
    -- Basic Information
    title VARCHAR(500) NOT NULL,
    description TEXT NOT NULL,
    risk_statement TEXT, -- "If [cause], then [event], resulting in [consequence]"
    
    -- Classification
    category_id UUID REFERENCES risk_categories(id) NOT NULL,
    sub_category_id UUID REFERENCES risk_categories(id),
    risk_type VARCHAR(50), -- 'inherent', 'residual', 'emerging', 'opportunity'
    
    -- Organizational Context
    business_unit_id UUID REFERENCES business_units(id),
    geography VARCHAR(100),
    process_area VARCHAR(200),
    
    -- Ownership
    risk_owner_id UUID REFERENCES users(id) NOT NULL,
    risk_delegate_id UUID REFERENCES users(id),
    
    -- Status & Lifecycle
    status VARCHAR(50) DEFAULT 'identified', 
    -- 'identified', 'assessed', 'mitigating', 'monitored', 'accepted', 'closed', 'retired'
    lifecycle_stage VARCHAR(50) DEFAULT 'active',
    -- 'emerging', 'active', 'declining', 'dormant', 'closed'
    
    -- Inherent Risk (before controls)
    inherent_likelihood INTEGER, -- 1-5 scale
    inherent_impact INTEGER, -- 1-5 scale
    inherent_risk_score INTEGER GENERATED ALWAYS AS (inherent_likelihood * inherent_impact) STORED,
    inherent_risk_rating VARCHAR(20), -- 'critical', 'high', 'medium', 'low', 'minimal'
    
    -- Residual Risk (after controls)
    residual_likelihood INTEGER,
    residual_impact INTEGER,
    residual_risk_score INTEGER GENERATED ALWAYS AS (residual_likelihood * residual_impact) STORED,
    residual_risk_rating VARCHAR(20),
    
    -- Target Risk (desired future state)
    target_likelihood INTEGER,
    target_impact INTEGER,
    target_risk_score INTEGER GENERATED ALWAYS AS (target_likelihood * target_impact) STORED,
    target_date DATE,
    
    -- Velocity & Proximity
    risk_velocity VARCHAR(20), -- 'immediate', 'fast', 'moderate', 'slow'
    risk_proximity VARCHAR(20), -- 'imminent', 'near_term', 'medium_term', 'long_term'
    
    -- Financial Quantification
    potential_loss_min DECIMAL(20,2),
    potential_loss_max DECIMAL(20,2),
    potential_loss_expected DECIMAL(20,2),
    currency VARCHAR(10) DEFAULT 'USD',
    annualized_loss_expectancy DECIMAL(20,2),
    
    -- Qualitative Impacts
    impact_financial TEXT,
    impact_operational TEXT,
    impact_regulatory TEXT,
    impact_reputational TEXT,
    impact_strategic TEXT,
    
    -- Causes & Consequences
    root_causes JSONB DEFAULT '[]',
    potential_consequences JSONB DEFAULT '[]',
    trigger_events JSONB DEFAULT '[]',
    
    -- Related Items
    related_risks JSONB DEFAULT '[]',
    related_controls JSONB DEFAULT '[]',
    related_kris JSONB DEFAULT '[]',
    related_incidents JSONB DEFAULT '[]',
    
    -- Regulatory Context
    applicable_regulations JSONB DEFAULT '[]',
    compliance_requirements JSONB DEFAULT '[]',
    
    -- Response Strategy
    risk_response VARCHAR(50), -- 'accept', 'mitigate', 'transfer', 'avoid'
    response_rationale TEXT,
    treatment_plan TEXT,
    
    -- Dates
    identified_date DATE DEFAULT CURRENT_DATE,
    last_assessed_date DATE,
    next_review_date DATE,
    
    -- Approval
    approved_by UUID REFERENCES users(id),
    approved_at TIMESTAMPTZ,
    approval_notes TEXT,
    
    -- Risk Appetite
    risk_appetite_id UUID,
    within_appetite BOOLEAN,
    appetite_breach_notes TEXT,
    
    -- Audit Trail
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_by UUID REFERENCES users(id),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    version INTEGER DEFAULT 1,
    
    -- AI Analysis
    ai_risk_score DECIMAL(5,2),
    ai_analysis JSONB,
    ai_recommendations JSONB,
    
    -- Metadata
    tags JSONB DEFAULT '[]',
    custom_fields JSONB DEFAULT '{}',
    attachments JSONB DEFAULT '[]'
);

-- Risk history for audit trail
CREATE TABLE risk_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    risk_id UUID REFERENCES risks(id) NOT NULL,
    field_changed VARCHAR(100) NOT NULL,
    old_value TEXT,
    new_value TEXT,
    change_reason TEXT,
    changed_by UUID REFERENCES users(id),
    changed_at TIMESTAMPTZ DEFAULT NOW()
);


-- =====================================================
-- RISK ASSESSMENTS

CREATE TABLE risk_assessment_templates (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    assessment_type VARCHAR(50), -- 'standard', 'detailed', 'rapid', 'scenario'
    
    -- Scoring Configuration
    likelihood_scale JSONB NOT NULL, 
    -- {"1": {"name": "Rare", "description": "< 1% chance"}, ...}
    impact_scale JSONB NOT NULL,
    -- {"1": {"name": "Minimal", "financial": "<10000"}, ...}
    
    risk_matrix JSONB NOT NULL,
    -- {"1-1": "minimal", "1-2": "low", ..., "5-5": "critical"}
    
    assessment_criteria JSONB DEFAULT '[]',
    requires_approval BOOLEAN DEFAULT true,
    approval_threshold INTEGER,
    escalation_threshold INTEGER,
    
    is_default BOOLEAN DEFAULT false,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE risk_assessments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    assessment_number VARCHAR(50) UNIQUE NOT NULL,
    
    -- Links
    risk_id UUID REFERENCES risks(id) NOT NULL,
    template_id UUID REFERENCES risk_assessment_templates(id),
    
    -- Assessment Type
    assessment_type VARCHAR(50) NOT NULL, -- 'initial', 'periodic', 'triggered', 'annual'
    trigger_reason TEXT,
    
    -- Status
    status VARCHAR(50) DEFAULT 'draft',
    -- 'draft', 'in_progress', 'pending_review', 'pending_approval', 'approved', 'rejected'
    
    -- Pre-Assessment (Inherent)
    inherent_likelihood INTEGER NOT NULL,
    inherent_likelihood_rationale TEXT,
    inherent_impact INTEGER NOT NULL,
    inherent_impact_rationale TEXT,
    inherent_score INTEGER GENERATED ALWAYS AS (inherent_likelihood * inherent_impact) STORED,
    inherent_rating VARCHAR(20),
    
    -- Control Effectiveness Assessment
    control_effectiveness VARCHAR(20),
    control_effectiveness_score INTEGER,
    control_gaps TEXT,
    
    -- Post-Assessment (Residual)
    residual_likelihood INTEGER NOT NULL,
    residual_likelihood_rationale TEXT,
    residual_impact INTEGER NOT NULL,
    residual_impact_rationale TEXT,
    residual_score INTEGER GENERATED ALWAYS AS (residual_likelihood * residual_impact) STORED,
    residual_rating VARCHAR(20),
    
    -- Trend
    trend VARCHAR(20), -- 'increasing', 'stable', 'decreasing'
    trend_rationale TEXT,
    
    -- Impact Details
    impact_financial JSONB,
    impact_operational JSONB,
    impact_regulatory JSONB,
    impact_reputational JSONB,
    impact_strategic JSONB,
    
    -- Velocity & Proximity
    velocity VARCHAR(20),
    velocity_rationale TEXT,
    proximity VARCHAR(20),
    proximity_rationale TEXT,
    
    -- Response Recommendation
    recommended_response VARCHAR(50),
    response_rationale TEXT,
    recommended_actions JSONB DEFAULT '[]',
    
    -- Assessment Details
    assessment_methodology TEXT,
    data_sources JSONB DEFAULT '[]',
    assumptions TEXT,
    limitations TEXT,
    
    -- Workflow
    assessor_id UUID REFERENCES users(id) NOT NULL,
    reviewer_id UUID REFERENCES users(id),
    approver_id UUID REFERENCES users(id),
    
    assessed_at TIMESTAMPTZ,
    reviewed_at TIMESTAMPTZ,
    review_notes TEXT,
    approved_at TIMESTAMPTZ,
    approval_notes TEXT,
    
    -- Scheduling
    assessment_period_start DATE,
    assessment_period_end DATE,
    next_assessment_date DATE,
    
    -- AI Assistance
    ai_assisted BOOLEAN DEFAULT false,
    ai_suggestions JSONB,
    ai_confidence DECIMAL(5,2),
    
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);


-- =====================================================
-- RISK QUANTIFICATION
-- =====================================================

CREATE TABLE risk_scenarios (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    scenario_number VARCHAR(50) UNIQUE NOT NULL,
    risk_id UUID REFERENCES risks(id) NOT NULL,
    
    -- Scenario Details
    name VARCHAR(200) NOT NULL,
    description TEXT,
    scenario_type VARCHAR(50), -- 'base_case', 'best_case', 'worst_case', 'stress_test'
    
    -- Loss Distribution Parameters
    distribution_type VARCHAR(50) DEFAULT 'lognormal',
    -- 'normal', 'lognormal', 'triangular', 'pert', 'uniform', 'custom'
    
    -- For triangular/PERT distributions
    loss_minimum DECIMAL(20,2),
    loss_most_likely DECIMAL(20,2),
    loss_maximum DECIMAL(20,2),
    
    -- For normal/lognormal distributions
    loss_mean DECIMAL(20,2),
    loss_std_dev DECIMAL(20,2),
    
    -- Frequency Parameters
    frequency_type VARCHAR(50) DEFAULT 'poisson',
    frequency_rate DECIMAL(10,4), -- Events per year
    frequency_n INTEGER,
    frequency_p DECIMAL(5,4),
    
    -- Correlation
    correlated_scenarios JSONB DEFAULT '[]',
    correlation_coefficient DECIMAL(5,4),
    
    -- Confidence & Assumptions
    confidence_level DECIMAL(5,2),
    data_quality VARCHAR(20),
    assumptions JSONB DEFAULT '[]',
    
    -- Monte Carlo Results
    simulation_runs INTEGER,
    simulated_at TIMESTAMPTZ,
    results_summary JSONB,
    -- {
    --   "mean": 150000,
    --   "median": 120000,
    --   "percentile_95": 300000,
    --   "percentile_99": 400000,
    --   "var_95": 300000,
    --   "cvar_95": 350000
    -- }
    results_distribution JSONB,
    
    is_active BOOLEAN DEFAULT true,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Loss event data (historical for calibration)
CREATE TABLE loss_events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    event_number VARCHAR(50) UNIQUE NOT NULL,
    
    risk_id UUID REFERENCES risks(id),
    category_id UUID REFERENCES risk_categories(id),
    business_unit_id UUID REFERENCES business_units(id),
    
    -- Event Details
    event_date DATE NOT NULL,
    discovered_date DATE,
    description TEXT NOT NULL,
    root_cause TEXT,
    
    -- Loss Details
    gross_loss DECIMAL(20,2) NOT NULL,
    recovered_amount DECIMAL(20,2) DEFAULT 0,
    net_loss DECIMAL(20,2) GENERATED ALWAYS AS (gross_loss - COALESCE(recovered_amount, 0)) STORED,
    currency VARCHAR(10) DEFAULT 'USD',
    
    -- Loss Type Breakdown
    loss_type VARCHAR(50),
    loss_breakdown JSONB,
    
    -- Insurance
    insurance_claim_amount DECIMAL(20,2),
    insurance_recovery DECIMAL(20,2),
    
    -- Status
    status VARCHAR(50) DEFAULT 'open',
    
    -- Related Items
    related_incident_id UUID,
    related_case_id UUID,
    
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE kri_definitions (

    calculation_formula TEXT,

    -- Ownership

    alert_recipients JSONB DEFAULT '[]',

CREATE TABLE kri_measurements (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    previous_value DECIMAL(20,4),

    change_percent DECIMAL(10,4),
    
    -- Status

    breach_duration INTEGER,
    
    -- Context
    notes TEXT,
    data_source TEXT,
    data_quality VARCHAR(20),
    
    -- Alert Generated
    alert_generated BOOLEAN DEFAULT false,
    alert_id UUID,
    
    recorded_by UUID REFERENCES users(id),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    
    UNIQUE(kri_id, measurement_date)
);
```

```
   ├── Reviewer validates

   └── Requests changes if needed
```

# 6. RISK QUANTIFICATION

## 6.1 Supported Distributions

| Distribution | Parameters | Best For |
|--------------|------------|----------|
| Triangular | Min, Most Likely, Max | Quick estimates |
| PERT | Min, Most Likely, Max | Reduced extremes |
| Lognormal | Mean, Std Dev | Operational risk |
| Normal | Mean, Std Dev | Market risk |

## 6.2 Monte Carlo Simulation

**Parameters:**
- Iterations: 10,000 (default), 100,000 (detailed)
- Frequency distribution: Poisson, Binomial, Fixed
- Loss distribution: Per scenario config

**Output Statistics:**
- Mean (Expected Loss)
- Median
- Standard Deviation
- Percentiles (5th, 25th, 50th, 75th, 90th, 95th, 99th)
- VaR at 95% and 99%
- Conditional VaR (Expected Shortfall)

## 6.3 Value at Risk (VaR)

**Methodologies:**
- Parametric VaR
- Historical VaR
- Monte Carlo VaR

**Portfolio VaR Formula:**
```
Portfolio VaR = √(Σ VaR_i² + 2 × Σ Σ ρ_ij × VaR_i × VaR_j)
```
