# Sentinel Counsel LLP - Background Research Feature

## Feature Overview

The Background Research feature provides AI-powered preliminary due diligence analysis for business transactions. It combines document analysis with web research to give attorneys a comprehensive picture of a target company, highlighting risks, contradictions, and key findings before deep-dive due diligence begins.

This is designed to be the **first thing an attorney runs** when starting work on a deal - a "quick intel briefing" that surfaces what matters most.

---

## USER FLOW

### Step 1: Create Research Request
User navigates to Background Research, clicks "New Research", and:
1. Selects an existing Deal (or creates standalone research)
2. Enters target company name and basic info
3. Uploads documents (or selects from Data Room)
4. Chooses which analysis modules to run
5. Kicks off the research

### Step 2: AI Processing
System processes documents and runs web searches in parallel:
- Document ingestion and text extraction
- Entity extraction (people, companies, amounts, dates)
- AI analysis across multiple dimensions
- Web search for external context
- Cross-reference and contradiction detection

### Step 3: Review Results
User receives a structured report with:
- Executive summary
- Detailed findings by category
- Risk indicators with severity ratings
- Cited sources (document page numbers, URLs)
- Recommended follow-up questions

```typescript
const COMPANY_OVERVIEW_PROMPT = `
You are a senior M&A attorney conducting preliminary due diligence on a target company.

Analyze the provided documents and create a comprehensive company overview including:

1. **Business Description**
   - What does the company do?
   - Core products/services
   - Value proposition
   - Business model (B2B, B2C, SaaS, manufacturing, etc.)

2. **Corporate History**
   - Founding date and founders
   - Key milestones and pivots
   - Major acquisitions or divestitures
   - Significant events

3. **Corporate Structure**
   - Parent company (if any)
   - Subsidiaries and affiliates
   - Ownership structure (if disclosed)
   - Geographic presence

4. **Operations Summary**
   - Headquarters location
   - Office/facility locations
   - Employee count (if mentioned)
   - Key operational metrics

For each finding, cite the specific document and page number where you found the information.
Flag any information that seems incomplete or requires verification.

Format your response as structured JSON.
`;
```

```typescript
Compare information across all provided documents to find:

3. **Timeline Inconsistencies**
   - Events dated differently
   - Sequence of events that don't align
   - Contracts with conflicting effective dates

4. **Statement Contradictions**
   - Management saying one thing in documents vs. another
   - External communications vs. internal communications
   - Marketing claims vs. actual capabilities
   - Projections vs. actuals (if historical projections available)

5. **Missing Information**
   - Topics mentioned in one document but absent from others
   - Incomplete explanations
   - References to documents not provided

For each contradiction found:
- Quote the specific conflicting statements
- Cite both source documents with page numbers
- Rate the significance (minor discrepancy vs. material misrepresentation)
- Suggest which version might be correct (if determinable)
- Recommend follow-up questions to resolve

This is CRITICAL for due diligence - contradictions often reveal problems.
`;
```
