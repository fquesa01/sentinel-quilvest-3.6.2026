# Video Conferencing Flexible Layout System - Implementation Spec

## Overview

Implement a flexible, resizable layout system for the video conferencing feature that allows users to dynamically adjust their view based on their current task. Legal professionals have different focus needs: a litigator taking a deposition may want to maximize transcript visibility, while someone observing witness reactions needs to maximize video. This system provides three primary view modes with seamless switching.

## State Management

interface VideoConferenceState {
  // View mode

## Resizable Panel Implementation

```tsx
// Drag handle for resizing panels
const ResizeHandle = ({ onMouseDown }) => (
  <div
    onMouseDown={onMouseDown}
    className="w-1 bg-slate-700/50 hover:bg-blue-500 cursor-col-resize flex items-center justify-center group transition-colors relative z-10"
  >
    {/* Larger invisible hit area */}
    <div className="absolute inset-y-0 -left-2 -right-2" />
    <GripVertical className="w-3 h-3 text-slate-600 group-hover:text-white" />
  </div>
);

// Resize logic
const handleMouseMove = useCallback((e) => {
  if (!isResizing || !containerRef.current) return;
  
  const containerRect = containerRef.current.getBoundingClientRect();
  const mouseX = e.clientX - containerRect.left;
  const percentage = (mouseX / containerRect.width) * 100;
  
  // Clamp between 25% and 80%
  const newTranscriptWidth = Math.max(25, Math.min(80, percentage));
  setPanelWidths({
    transcript: newTranscriptWidth,
    ai: 100 - newTranscriptWidth
  });
}, [isResizing]);
```

## Video Layout Toggle (Grid/Speaker)

```tsx
const VIDEO_LAYOUTS = {
  grid: { name: 'Grid', icon: Grid },
  speaker: { name: 'Speaker', icon: User },
};

const VideoLayoutToggle = ({ currentLayout, onChange, visible }) => {
  if (!visible) return null;
  
  return (
    <div className="flex items-center bg-slate-700 rounded-lg p-0.5">
      {Object.entries(VIDEO_LAYOUTS).map(([key, layout]) => (
        <button
          key={key}
          onClick={() => onChange(key)}
          className={`p-1.5 rounded transition-colors ${
            currentLayout === key 
              ? 'bg-slate-600 text-white' 
              : 'text-slate-400 hover:text-slate-200'
          }`}
          title={layout.name}
        >
          <layout.icon className="w-3.5 h-3.5" />
        </button>
      ))}
    </div>
  );
};
```

## Integration Points

### Ambient Intelligence Integration
- When ambient AI is enabled, the AI Suggestions panel should auto-update based on transcript content
- The "Ambient AI" badge in top bar indicates feature is active
- AI suggestions come from the existing RAG system querying case documents

### Case/Transaction Linking
- Meeting should be linkable to a case (existing functionality)
- When linked, AI suggestions filter to documents from that case
- Transcript is auto-associated with the case for later review

### Recording Integration
- Recording indicator (red dot) in top bar when active
- Transcript is tied to recording timestamps
- Video Focus mode should still show recording indicator prominently

## Styling Guidelines

Consistent with Sentinel's existing dark theme:
